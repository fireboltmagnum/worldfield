# Day 9 Research Agenda Report

## Failure Analysis

Phase-1 question answers:

1. **Is Day 9b actually evaluating joint conditioning?**
   - Candidate joint sets are generated by `combinations(others, r)` for `r=1..3`.
   - So: **yes, joint sets are generated syntactically**.

2. **Is it only evaluating variables independently?**
   - In the actual current implementation, effectively **yes for the failed runs**.
   - Reason: every correct joint explanatory set in double/triple confound is rejected by the pre-gate
     `base_ij >= 0.95 * weakest_link` before `I(A;B|S)` is allowed to influence the score.

3. **Are candidate explanatory sets being generated correctly?**
   - **Yes.** The correct sets `{C,D}` and `{C,D,E}` appear in every failed-run trace.

4. **Are explanatory sets generated but rejected incorrectly?**
   - **Yes.** This is the dominant root cause in the current code path.
   - Double-confound failed runs: correct joint set skipped by gate in 50/50.
   - Triple-confound failed runs: correct joint set skipped by gate in 50/50.

5. **Does the scoring rule fail even when the correct explanatory set is tested?**
   - Diagnostic answer: **also yes, in the naive ungated family**.
   - When the gate is removed as a diagnostic, the family becomes unstable and over-suppresses true edges:
     - confound: 7/20
     - double_confound: 3/20
     - triple_confound: 3/20
     - chain: 4/20
     - collider: 2/20

### Root Cause

The Day 9b failure is **not** just 'multi-confounds are impossible' and **not** just 'the set learner was never called'.
It is a two-part failure:

1. **Implementation limitation:** the current set learner generates the correct joint sets but the weakest-link pre-gate rejects them before scoring.
2. **Family/scoring limitation:** if that gate is removed naively, the current suppression rule over-suppresses true edges and breaks even previously-passing worlds.

So the precise boundary is:
- the benchmark worlds contain the right conditional-information signal
- the current PMI + screening heuristic cannot use that signal safely

# Day 9 Benchmark Validation

## Single Confound
Diagram:
```text
magenta diamond <- crimson square -> skyblue pentagon
blue square -- teal pentagon   (direct control)
```
Expected undirected skeleton:
- keep `magenta diamond--crimson square`
- keep `skyblue pentagon--crimson square`
- remove `magenta diamond--skyblue pentagon`
- keep `blue square--teal pentagon`

Why:
- `A` and `B` are d-separated by conditioning on `C`.
- The direct control edge should remain because no explanatory parent screens it off.

## Double Confound
Diagram:
```text
magenta diamond <- crimson square -> skyblue pentagon
magenta diamond <- red circle -> skyblue pentagon
blue square -- teal pentagon
```
Expected undirected skeleton:
- keep `magenta diamond--crimson square`, `skyblue pentagon--crimson square`, `magenta diamond--red circle`, `skyblue pentagon--red circle`
- remove `magenta diamond--skyblue pentagon`
- keep `blue square--teal pentagon`

Why:
- `A` and `B` remain associated after conditioning on only `C` or only `D`.
- `A` and `B` should disappear only after conditioning on the JOINT set `{crimson square, red circle}`.

## Triple Confound
Diagram:
```text
magenta diamond <- crimson square -> skyblue pentagon
magenta diamond <- red circle -> skyblue pentagon
magenta diamond <- blue square -> skyblue pentagon
teal pentagon -- olive square
```
Expected undirected skeleton:
- keep `magenta diamond--crimson square`, `skyblue pentagon--crimson square`, `magenta diamond--red circle`, `skyblue pentagon--red circle`, `magenta diamond--blue square`, `skyblue pentagon--blue square`
- remove `magenta diamond--skyblue pentagon`
- keep `teal pentagon--olive square`

Why:
- `A` and `B` should disappear only after conditioning on the JOINT set `{crimson square, red circle, blue square}`.

## Chain
Diagram:
```text
magenta diamond -> skyblue pentagon -> crimson square
blue square -- teal pentagon
```
Expected undirected skeleton:
- keep `magenta diamond--skyblue pentagon`
- keep `skyblue pentagon--crimson square`
- remove `magenta diamond--crimson square`
- keep `blue square--teal pentagon`

Why:
- In a chain, the endpoints become independent when conditioning on the middle node.

## Collider
Diagram:
```text
magenta diamond -> crimson square <- skyblue pentagon
blue square -- teal pentagon
```
Expected undirected skeleton:
- keep `magenta diamond--crimson square`
- keep `skyblue pentagon--crimson square`
- remove `magenta diamond--skyblue pentagon` marginally
- keep `blue square--teal pentagon`

Why:
- Marginally, `A` and `B` are independent in a collider.
- Conditioning on `C` OPENS the path, so `I(A;B|C)` should increase.

## Can The Current Family Work?

### Single confound
- **One-variable screen:** yes in principle and in practice, because `I(A;B|C)` is near zero and a single parent explains the correlation.
- **Set-conditioned screen:** yes in principle and in practice.

### Double confound
- **One-variable screen:** no in principle. Conditioning on only `C` or only `D` leaves residual dependence, so the family cannot solve this benchmark.
- **Set-conditioned screen:** yes in principle, because `I(A;B|{C,D})` is near zero in the diagnostic measurements.
- **Current implementation:** no in practice, because the pre-gate blocks the correct set and the naive ungated score destabilizes the family.

### Triple confound
- **One-variable screen:** no in principle.
- **Set-conditioned screen:** yes in principle, because `I(A;B|{C,D,E})` is near zero in the diagnostic measurements.
- **Current implementation:** no in practice, for the same two-part reason as double confound.

### Major Milestone

The direct conditional-information diagnostics show that the benchmark is not flawed and the signal is not absent.
Therefore the next problem is genuinely:

> Can a graph built from experience represent causal structure strongly enough to use higher-order conditional explanations without collapsing true edges?

That is beyond the old Day-8 question of whether WorldField merely learns association.
